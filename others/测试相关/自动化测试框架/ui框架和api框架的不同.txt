测试对象
UI 框架：测试对象是用户界面，主要验证页面元素的显示、交互逻辑和用户操作的响应。
API 框架：测试对象是应用程序的接口，主要验证接口的功能、性能和安全性。
依赖环境
UI 框架：依赖于浏览器和前端页面，需要安装浏览器驱动，测试环境相对复杂。
API 框架：只需要依赖于网络环境，测试环境相对简单。
测试速度
UI 框架：由于需要加载页面和模拟用户操作，测试速度相对较慢。
API 框架：直接调用接口，不需要加载页面，测试速度相对较快。
重难点
UI 框架
元素定位：页面元素的定位可能会受到页面布局、元素属性变化等因素的影响，需要使用多种定位方式和等待机制来确保元素的准确定位。
页面加载和渲染：页面加载和渲染时间可能会影响测试的稳定性，需要使用等待机制来确保页面元素完全加载和渲染。
浏览器兼容性：不同浏览器对页面的渲染可能会存在差异，需要在不同浏览器上进行测试，确保测试的兼容性。
API 框架
接口依赖：有些接口可能依赖于其他接口的返回结果，需要处理接口之间的依赖关系，确保测试用例的独立性和可重复性。
数据处理：接口返回的数据可能是复杂的 JSON 或 XML 格式，需要对数据进行解析和处理，确保数据的正确性和完整性。
接口安全性：需要对接口的安全性进行测试，如验证接口的认证、授权机制，防止接口被恶意攻击。
生成 Allure 报告
在项目根目录下执行以下命令生成 Allure 报告：
bash
Copy
pytest --alluredir=./allure-results
allure serve ./allure-results
