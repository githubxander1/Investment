代码思路总结
这段代码实现了一个简单的文件上传和管理应用，使用 Flask 框架。以下是代码的主要功能和逻辑：
设置上传文件夹：
配置上传文件夹为 uploads，并确保该文件夹存在。
处理默认路由：
默认路由 / 返回 index.html 页面。
上传文件：
POST 请求 /upload_file 接收上传的文件，并保存到指定文件夹。
使用 secure_filename 确保文件名安全。
上传文本：
POST 请求 /upload_text 接收上传的文本，并保存为 .txt 文件。
获取已上传文件列表：
GET 请求 /get_files 返回已上传文件的列表，包括文件名、大小和上传时间。



展示已上传文件：
GET 请求 /uploads/<filename> 返回指定文件。
写作思路逻辑：
定义路由：
使用 @app.route('/uploads/<filename>') 定义动态路由。
发送文件：
使用 send_from_directory 发送指定文件。

删除文件：
DELETE 请求 /delete/<filename> 删除指定文件。
处理 favicon 请求：
GET 请求 /favicon.ico 返回 favicon 图标。
关于Favicon:•Favicon（Favorites Icon的缩写，也称作网站图标、书签图标、标签图标）是一个网站或网页在浏览器的地址栏、书签列表以及多标签页浏览器的标签页中显示的小型图标。它的作用主要是为了增加品牌辨识度，使用户能够在众多的标签页中快速找到特定的网站。
对新手开发者的必要基础知识点
1. Flask 基础
Flask 框架：轻量级 Web 框架，适合快速开发。
创建 Flask 应用：通过 Flask(__name__) 创建应用实例。
路由装饰器：使用 @app.route() 装饰器定义路由。
请求处理：使用 request 对象处理 HTTP 请求。
2. 文件上传
处理文件上传：使用 request.files 获取上传的文件。
保存文件：使用 secure_filename 确保文件名安全，然后使用 file.save() 保存文件。
文件路径操作：使用 os.path.join() 构建文件路径。
3. JSON 响应
返回 JSON 数据：使用 jsonify() 将 Python 字典转换为 JSON 格式响应。
4. 文件操作
读写文件：使用 open() 打开文件，进行读写操作。
文件属性：使用 os.path.getsize() 获取文件大小，使用 os.path.getmtime() 获取修改时间。
5. 目录操作
创建目录：使用 os.makedirs() 创建目录。
列出目录内容：使用 os.listdir() 列出目录下的文件。
6. 时间处理
格式化时间：使用 datetime.fromtimestamp() 将时间戳转换为 datetime 对象，并使用 strftime() 格式化时间。
7. HTTP 方法
处理不同方法：使用不同的 HTTP 方法（GET、POST、DELETE）处理不同的请求。

发送静态文件：使用 send_from_directory() 发送静态文件。
处理 favicon 请求：使用 favicon 路由处理 favicon 请求。
9. 运行 Flask 应用
启动应用：使用 app.run启动flask，指定主机和端口

js知识点总结：
event.preventDefault(); 是JavaScript中一个常用的方法，主要用于阻止事件的默认行为。当你将此方法绑定到特定事件（如表单的submit事件）的事件处理器中时，它会阻止该事件的默认动作发生。具体到表单提交的场景中，它的作用主要包括：1. 阻止刷新页面：默认情况下，当用户提交一个HTML表单时，页面会刷新或跳转到表单action属性指定的URL。使用event.preventDefault();可以阻止这种自动刷新或跳转行为。2. 保留用户输入：因为页面不再刷新，用户在表单中输入的数据得以保留，这对于需要通过Ajax异步提交表单或者进行客户端验证的场景尤为重要。3. 允许自定义处理：这使得开发者可以在阻止默认提交行为后，执行自定义的逻辑，比如使用Ajax发送表单数据、显示验证错误信息等，从而提供更好的用户体验和控制流程。例如，在处理表单提交事件时，你可能会这样使用：document.getElementById("myForm").addEventListener("submit", function(event) {
    event.preventDefault(); // 阻止表单默认提交
    // 在这里可以添加自定义的表单处理逻辑
});
